# Supabase Functions (Microservice Layer)

This folder is the dedicated edge microservice workspace for tenant-shop.

## Structure

- `_shared/`: reusable enterprise utilities (logger, env parser, crypto)
- `user-created-notify/`: function for auth user-created notifications to admin/superadmin

## First Function

`user-created-notify`

- TypeScript edge function
- Receives secure internal trigger payload
- Loads `app_settings` SMTP fields
- Decrypts SMTP secret from `smtp_password_iv` and `smtp_password_content` with `CRYPTO_KEY`
- Sends email to all active `superadmin` and `admin` recipients
- Uses structured JSON logs

## Required Secrets

Use environment-specific secret file and deploy script:

```bash
cp .env.production.example .env.production
# fill values
pnpm deploy:user-created-notify:production
```

Function-owned secrets to define in `functions/.env.<env>`:

- `CRYPTO_KEY`
- `INTERNAL_EDGE_TRIGGER_TOKEN`
- `APP_BASE_URL` (optional)
- `SMTP_TIMEOUT_MS` (optional)

Notes:

- `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are runtime-provided in deployed Supabase Edge Functions.
- For local `supabase functions serve`, if your setup does not auto-provide them, export them in your shell before `pnpm serve` (do not commit them in `*.example`).

Project ref resolution order for deploy:

1. shell env: `SUPABASE_PROJECT_REF`
2. env file key: `SUPABASE_PROJECT_REF=...`
3. derived from `SUPABASE_URL` when present (e.g. `https://abcxyz.supabase.co` => `abcxyz`)

## Trigger Wiring

A DB trigger migration calls this function through `pg_net`.

In Supabase managed Postgres, custom `ALTER DATABASE ... SET` may be blocked.
So runtime trigger settings are stored in `public.internal_edge_settings`.

Configure with SQL (one-time per environment):

```sql
INSERT INTO public.internal_edge_settings (id, edge_functions_base_url, edge_functions_token)
VALUES (1, 'https://<project-ref>.supabase.co/functions/v1', '<long-random-token>')
ON CONFLICT (id)
DO UPDATE SET
  edge_functions_base_url = EXCLUDED.edge_functions_base_url,
  edge_functions_token = EXCLUDED.edge_functions_token,
  updated_at = now();
```

## Logging

All functions should log via `_shared/logger.ts` in JSON format:

- `ts`
- `level`
- `message`
- `context`

This is required for enterprise observability consistency.
